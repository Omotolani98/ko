■ Core Concepts

Services.

A service is the fundamental unit of organization in Kọ́. It represents a logical microservice boundary — a group of related APIs, databases, and infrastructure that belong together.

Defining a Service

Annotate any class with @KoService to declare it as a service. The value is a kebab-case name that uniquely identifies the service across your application. This name appears in the Application Model, service discovery, tracing, and the dev dashboard.

OrderService.java Java 
@KoService("order-service")
public class OrderService {

    @KoDatabase(name = "orders")
    private KoSQLDatabase db;

    @KoPubSub(topic = "order-placed")
    private KoTopic<OrderPlacedEvent> orderTopic;

    @KoAPI(method = "POST", path = "/orders", auth = true)
    public OrderResponse placeOrder(PlaceOrderRequest req) {
        // ...
    }

    @KoCron(schedule = "0 */6 * * *", name = "expire-pending-orders")
    public void expirePendingOrders() {
        // ...
    }
}

Naming Rules

Service names must follow kebab-case convention: lowercase letters and digits separated by hyphens. The name is validated at compile time — invalid names fail the build.

NameValid?Reason
user-serviceYesKebab-case with letters
auth2YesSingle word with digit
order-service-v2YesMultiple segments
UserServiceNoUppercase not allowed
user_serviceNoUnderscores not allowed
-serviceNoCannot start with hyphen

What a Service Contains

→

API Endpoints

Methods annotated with @KoAPI — the HTTP surface of the service.

◫

Infrastructure

Fields annotated with @KoDatabase, @KoPubSub, @KoCache, etc.

↻

Background Work

Cron jobs (@KoCron) and subscribers (@KoSubscribe).
