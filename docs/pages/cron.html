Cron Jobs.

Schedule recurring work with @KoCron. Validated at compile time.

Cron job examples Java 
// Every day at 3 AM
@KoCron(schedule = "0 3 * * *", name = "daily-cleanup")
public void dailyCleanup() { /* ... */ }

// Every 15 minutes
@KoCron(schedule = "*/15 * * * *", name = "sync-inventory")
public void syncInventory() { /* ... */ }

// Weekdays at 9 AM
@KoCron(schedule = "0 9 * * MON-FRI", name = "send-digest")
public void sendDailyDigest() { /* ... */ }

Cron Expression Format

Kọ́ uses standard 5-field cron expressions. The processor validates these at compile time — invalid expressions fail the build.

Cron format Text 
┌────────────── minute (0-59)
│ ┌──────────── hour (0-23)
│ │ ┌────────── day of month (1-31)
│ │ │ ┌──────── month (1-12 or JAN-DEC)
│ │ │ │ ┌────── day of week (0-7 or MON-SUN)
│ │ │ │ │
* * * * *

Note

The smallest granularity is 1 minute. There is no seconds field in Kọ́ cron expressions. Use */1 * * * * or * * * * * for every-minute execution.

How It Works

At startup, the runtime reads @KoCron methods from the Application Model, parses the cron expression, and schedules each job. Jobs run on virtual threads and automatically reschedule after each execution. The scheduler computes the next execution time from the cron expression, so jobs always fire at the correct wall-clock time.

Compile-Time Validation

Invalid cron expressions like "60 * * * *" (minute > 59) or "* * *" (wrong field count) will cause a build failure with a descriptive error message.
