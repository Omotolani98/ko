Secrets.

Declare application secrets with @KoSecret. Resolved from environment variables locally, from secret managers in production.

Using secrets Java 
@KoService("payment-service")
public class PaymentService {

    @KoSecret("stripe-api-key")
    private KoSecretValue stripeKey;

    @KoSecret("webhook-signing-secret")
    private KoSecretValue webhookSecret;

    @KoAPI(method = "POST", path = "/payments", auth = true)
    public PaymentResponse charge(ChargeRequest req) {
        Stripe.init(stripeKey.requireValue());
        // ...
    }
}

Annotation Attributes

AttributeTypeDefaultDescription

API Methods

MethodDescription

value()Resolve and return the secret value (null if not set)
requireValue()Resolve and return the secret value, throwing if not set

Environment Variable Mapping

Locally, secret names are converted to UPPER_SNAKE_CASE environment variables. For example, "stripe-api-key" maps to STRIPE_API_KEY. In production, a different KoSecretProvider routes to AWS Secrets Manager, GCP Secret Manager, or HashiCorp Vault.

Tip

Use requireValue() for secrets that must be present at runtime — it throws an IllegalStateException with a clear message if the secret is missing. Use value() for optional secrets where a fallback is acceptable.
