Databases.

Declare SQL databases with @KoDatabase. The framework provisions them locally with H2 and connects to managed instances in production.

Declaring a database Java 
@KoService("inventory-service")
public class InventoryService {

    @KoDatabase(name = "inventory")
    private KoSQLDatabase db;

    @KoAPI(method = "GET", path = "/stock/:sku")
    public Map<String, Object> getStock(@PathParam("sku") String sku) {
        return db.queryRow(
            "SELECT * FROM stock WHERE sku = ?", sku
        );
    }

    @KoAPI(method = "GET", path = "/stock")
    public List<Map<String, Object>> listStock() {
        return db.query("SELECT * FROM stock ORDER BY sku");
    }

    @KoAPI(method = "POST", path = "/stock")
    public void addStock(StockRequest req) {
        db.exec("INSERT INTO stock (sku, quantity) VALUES (?, ?)",
            req.sku(), req.quantity());
    }
}

Annotation Attributes

AttributeTypeDefaultDescription
nameString—Unique database name (kebab-case)
migrationsString"migrations"Classpath path to SQL migration files

API Methods

MethodDescription
query(sql, params...)Execute a query and return all rows as List<Map<String, Object>>
queryRow(sql, params...)Execute a query and return the first row (or null)
exec(sql, params...)Execute an INSERT/UPDATE/DELETE, returns affected row count
tx(action)Execute multiple statements within a transaction

Transactions

Using transactions Java 
db.tx(tx -> {
    tx.exec("UPDATE accounts SET balance = balance - ? WHERE id = ?", amount, fromId);
    tx.exec("UPDATE accounts SET balance = balance + ? WHERE id = ?", amount, toId);
});

Migrations

SQL migration files are automatically run at startup. Place them at src/main/resources/migrations/{database-name}/ and name them with numbered prefixes for ordering.

migrations/inventory/001_create_stock.sql SQL 
CREATE TABLE IF NOT EXISTS stock (
    id        BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    sku       VARCHAR(100) NOT NULL UNIQUE,
    quantity  INT NOT NULL DEFAULT 0
);
Tip

Migration files are executed in alphabetical order. Use numbered prefixes like 001_, 002_ to control the order.

Local Development

Locally, each @KoDatabase gets an H2 in-memory database in PostgreSQL compatibility mode. No Docker needed. SQL migrations run automatically at startup.
