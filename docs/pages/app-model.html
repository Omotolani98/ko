What It Contains

S

Services

Name, class, package, and all contained APIs, databases, pub/sub topics, caches, cron jobs, secrets, and buckets.

T

Types

Full field-level schema for every request and response record, including nested types.

D

Dependencies

Service-to-service dependencies — API calls, shared pub/sub topics, and their direction.

I

Infrastructure

Aggregate list of all databases, pub/sub topics, caches, and other resources needed by the application.

Schema Structure

ko-app-model.json (schema) JSON 
{
  "schema": "1.0",                  // Model schema version
  "version": "0.1.0-SNAPSHOT",     // App version
  "app": "my-app",                 // Application name
  "generated_at": "2026-...",      // ISO-8601 timestamp
  "services": [                    // All discovered services
    {
      "name": "...",
      "class_name": "...",
      "package_name": "...",
      "apis": [/* ApiEndpointModel[] */],
      "databases": [/* DatabaseModel[] */],
      "publishes": [/* topic names */],
      "subscribes": [/* topic names */],
      "caches": [/* CacheModel[] */],
      "cron_jobs": [/* CronJobModel[] */],
      "secrets": [/* SecretModel[] */],
      "buckets": [/* BucketModel[] */]
    }
  ],
  "pubsub_topics": [/* PubSubTopicModel[] */],
  "databases": [/* DatabaseModel[] */],
  "service_dependencies": [/* ServiceDependencyModel[] */]
}
Design Principle

The Application Model uses snake_case for all JSON field names. This is configured via Jackson's PropertyNamingStrategies.SNAKE_CASE on the ObjectMapper, so Java records use standard camelCase field names while the JSON output is consistently snake_case.
