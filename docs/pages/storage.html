Object Storage.

Declare storage buckets with @KoBucket. Locally backed by the filesystem, in production by S3, GCS, or compatible stores.

Using a bucket Java 
@KoBucket(name = "user-avatars", publicRead = true)
private KoBucketStore avatarBucket;

@KoAPI(method = "POST", path = "/users/:id/avatar", auth = true)
public AvatarResponse uploadAvatar(
    @PathParam("id") String userId,
    UploadRequest req
) {
    String key = "avatars/" + userId + ".png";
    avatarBucket.upload(key, req.data(), "image/png");
    return new AvatarResponse(key);
}

Annotation Attributes

AttributeTypeDefaultDescription
nameString—Unique bucket name (kebab-case)
publicReadbooleanfalseWhether objects are publicly readable

API Methods

MethodDescription
upload(key, data, contentType)Upload a byte array to the bucket
download(key)Download an object as a byte array (null if not found)
delete(key)Delete an object from the bucket
list(prefix)List object keys matching a prefix
list()List all object keys in the bucket
exists(key)Check if an object exists

Tip

Each @KoBucket maps to a directory locally. Files are stored at ko-storage/{bucket-name}/{key} in your system's temp directory. In production, a different KoStorageProvider routes to S3, GCS, or another object store.
