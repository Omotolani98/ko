>_ Reference

CLI Reference.

The ko command-line tool orchestrates local development, builds, and code generation.

Terminal Shell 
# Create a new project
$ ko app --name my-cool-app

# Start local dev environment
$ ko run

# Start on a specific port
$ ko run --port=9000

# Run with file watching and Docker containers
$ ko run --watch --containers

# Run without Docker (in-memory providers)
$ ko run --containers=false

# Check daemon status
$ ko status

# Build production Docker image
$ ko build docker --config infra-prod.json my-app:latest

# Generate TypeScript API client
$ ko gen client --lang typescript --output ./frontend/src/api

# Run tests with isolated infrastructure
$ ko test

# View the application model
$ ko model

# Open the dev dashboard
$ ko dashboard

ko app

Scaffold a new Kọ́ project with Gradle build, Spring Boot configuration, a sample service with @KoService and @KoAPI annotations, and a test class using ko-test.

FlagShortDefaultDescription
--nameApplication name in kebab-case (required)
--packagecom.example.<appname>Java package for the generated code
Terminal Shell 
# Create a new app with default package
$ ko app --name my-cool-app

# Create with a custom package
$ ko app --name my-cool-app --package com.mycompany.coolapp

ko run

Build the application model, provision infrastructure containers, generate infra-config.json, start the dev dashboard, and run the Spring Boot app via Gradle bootRun. File changes are watched and trigger automatic rebuilds.

FlagShortDefaultDescription
--port-p8080HTTP port for the application
--module-mGradle module to run (e.g., :examples:hello-world)
--dashboard-port9400Port for the dev dashboard
--watchtrueWatch source files and rebuild on changes
--containerstrueUse Docker containers for infrastructure (Postgres, Redis, Kafka, MinIO)
Terminal Shell 
# Run on the default port (8080)
$ ko run

# Run on a custom port
$ ko run --port=9000

# Run a specific module
$ ko run -m :examples:hello-world -p 3000

# Run without Docker containers (in-memory providers)
$ ko run --containers=false

# Run without file watching
$ ko run --watch=false

ko status

Show the status of the running Ko daemon. Connects to ~/.ko/daemon.sock and displays the daemon state, uptime, ports, running containers, and file watcher status.

Terminal Shell 
# Check daemon status
$ ko status
