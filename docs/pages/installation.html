⚙ Getting Started

Installation.

Requirements and setup for using Kọ́ in your Java project.

Requirements

☘

Java 21+

Kọ́ uses records, sealed interfaces, pattern matching, and virtual threads.

⚙

Gradle 8.12+

The Kọ́ Gradle plugin requires Gradle 8.12 or later with version catalogs support.

▦

Docker (optional)

Required for ko run with Testcontainers. Not needed for compile-time features.

Gradle Setup

settings.gradle.kts Kotlin DSL 
pluginManagement {
    repositories {
        mavenCentral()
        gradlePluginPortal()
    }
}

rootProject.name = "my-project"
build.gradle.kts Kotlin DSL 
plugins {
    id("dev.ko") version "0.1.0-SNAPSHOT"
}

ko {
    appName.set("my-app")
}

java {
    toolchain {
        languageVersion.set(JavaLanguageVersion.of(21))
    }
}

Version Catalog (optional)

If you manage dependencies via gradle/libs.versions.toml, add the Kọ́ entries alongside your other dependencies.

gradle/libs.versions.toml TOML 
[versions]
ko = "0.1.0-SNAPSHOT"

[libraries]
ko-annotations = { module = "dev.ko:ko-annotations", version.ref = "ko" }
ko-processor   = { module = "dev.ko:ko-processor",   version.ref = "ko" }
ko-runtime     = { module = "dev.ko:ko-runtime",     version.ref = "ko" }
ko-test        = { module = "dev.ko:ko-test",        version.ref = "ko" }

[plugins]
ko = { id = "dev.ko", version.ref = "ko" }
