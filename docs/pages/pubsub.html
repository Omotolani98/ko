Pub/Sub.

Declare message topics with @KoPubSub and subscribe with @KoSubscribe. The framework handles serialization, delivery guarantees, retries, and dead-letter queues.

Publishing

Publishing events Java 
@KoPubSub(topic = "order-placed", delivery = DeliveryGuarantee.AT_LEAST_ONCE)
private KoTopic<OrderPlacedEvent> orderTopic;

// Later in your code...
orderTopic.publish(new OrderPlacedEvent(order.id(), order.total()));

Subscribing

Subscribing to events Java 
@KoSubscribe(topic = "order-placed", name = "send-confirmation",
             maxRetries = 5, deadLetter = true)
public void onOrderPlaced(OrderPlacedEvent event) {
    emailService.sendConfirmation(event.orderId());
}

Delivery Guarantees

GuaranteeBehaviorUse Case
AT_LEAST_ONCEMessage delivered one or more times. Subscriber must be idempotent.Most use cases (default)
AT_MOST_ONCEMessage delivered zero or one times. May be lost on failure.Analytics, non-critical notifications
EXACTLY_ONCEMessage delivered exactly once (transactional where supported).Financial transactions, inventory
