⚡ Infrastructure

Caching.

Declare named caches with @KoCache. Locally backed by ConcurrentHashMap, in production by Redis or Memcached.

Using a cache Java 
@KoCache(name = "product-cache", ttl = 900)
private KoCacheCluster<String, ProductResponse> cache;

@KoAPI(method = "GET", path = "/products/:id")
public ProductResponse getProduct(@PathParam("id") String id) {
    return cache.getOrSet(id, () ->
        loadProductFromDb(id)
    );
}

Annotation Attributes

AttributeTypeDefaultDescription
nameString—Unique cache name (kebab-case)
ttllong300Time-to-live in seconds

API Methods

MethodDescription
get(key)Get a cached value (returns null if not found or expired)
getOrSet(key, loader)Get a cached value, or compute and cache it if missing
set(key, value)Store a value with the default TTL
set(key, value, ttlSeconds)Store a value with a custom TTL
delete(key)Remove a value from the cache

Tip

The local in-memory cache respects TTL — expired entries are evicted on access. Use getOrSet() as your default pattern to avoid cache stampedes.
