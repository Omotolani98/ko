▶ Getting Started

Quickstart.

Get a Kọ́ service running in under five minutes. You'll define a service, add an API endpoint and a database, then watch the framework wire everything together.

Create a new Gradle project

Use the standard Gradle init or your IDE to scaffold a Java 21 project.

Apply the Kọ́ Gradle plugin

The plugin adds annotation processing and model generation to your build.

Define your first service

Annotate a class with @KoService and add methods with @KoAPI.

Build and inspect the model

Run ./gradlew build and find ko-app-model.json in your build output.

1. Apply the plugin

build.gradle.kts Kotlin DSL 
plugins {
    id("dev.ko")
}

ko {
    appName.set("my-app")
}

dependencies {
    implementation("dev.ko:ko-annotations:0.1.0-SNAPSHOT")
    annotationProcessor("dev.ko:ko-processor:0.1.0-SNAPSHOT")
}
Tip

The Kọ́ Gradle plugin automatically adds the annotation and processor dependencies, sets the -parameters compiler flag, and registers the model generation task.

2. Write your first service

GreetingService.java Java 
package com.example;

import dev.ko.annotations.*;

@KoService("greeting-service")
public class GreetingService {

    /** Say hello to someone. */
    @KoAPI(method = "GET", path = "/hello/:name")
    public GreetingResponse hello(@PathParam("name") String name) {
        return new GreetingResponse("Hello, " + name + "!");
    }
}

public record GreetingResponse(String message) {}

3. Build

Terminal Shell 
$ ./gradlew build

What just happened?

The annotation processor scanned your @KoService class at compile time, extracted the API endpoints, request/response types, and Javadoc, then serialized everything into a structured Application Model. This model is the foundation for runtime wiring, documentation generation, and the dev dashboard.
