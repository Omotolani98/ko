Service-to-Service Calls.

Call other services with type-safe generated clients. Same code works as in-process calls locally and HTTP calls in production.

Using a service client Java 
@KoService("notification-service")
public class NotificationService {

    @KoServiceClient
    private UserServiceClient userService;  // Auto-generated

    @KoSubscribe(topic = "order-placed", name = "notify-user")
    public void onOrderPlaced(OrderPlacedEvent event) {
        // Type-safe call to user-service — no HTTP boilerplate
        UserResponse user = userService.getUser(event.userId());
        sendEmail(user.email(), "Your order has been placed!");
    }
}
How service calls are routed
Service Client
→
KoServiceCaller
↓
Local: InProcessCaller
Direct method invocation
Prod: HttpServiceCaller
HTTP via service discovery

The KoServiceCaller interface routes calls based on the environment. Locally, when all services run in a single JVM, InProcessCaller invokes the target method directly — no network hop. In production, HttpServiceCaller resolves the target URL from service_discovery in infra-config.json and makes an HTTP call. Your code never changes.
