Service-to-Service Calls.

Call other services with type-safe generated clients. Same code works as in-process calls locally and HTTP calls in production.

Defining a service Java 
@KoService("greeting-service")
public class GreetingService {

    @KoAPI(method = "GET", path = "/hello/:name")
    public GreetingResponse hello(@PathParam("name") String name) {
        return new GreetingResponse("Hello, " + name + "!");
    }
}
Calling it from another service Java 
@KoService("stats-service")
public class StatsService {

    @KoServiceClient
    private GreetingServiceClient greetingClient;  // Auto-generated at compile time

    @KoAPI(method = "GET", path = "/stats/hello/:name")
    public GreetingResponse proxyHello(@PathParam("name") String name) {
        return greetingClient.hello(name);
    }
}

Generated Client

For each @KoService, the annotation processor generates a {ServiceName}Client class at compile time. Each @KoAPI method becomes a fully typed client method with matching parameters and return type.

Generated: GreetingServiceClient.java Java (auto-generated) 
@Generated("ko-annotation-processor")
public class GreetingServiceClient {
    private KoServiceCaller caller;

    public GreetingResponse hello(String name) {
        return (GreetingResponse) caller.call(
            "greeting-service", "GET", "/hello/:name",
            new Object[]{name}
        );
    }
    // ... one method per @KoAPI endpoint
}

Tip

Locally, InProcessCaller invokes the target method directly in the same JVM — no serialization, no HTTP, no network hop. The generated client gives you compile-time type safety while the caller handles routing transparently.

Production Routing

In production, HttpServiceCaller resolves the target service URL from service_discovery in infra-config.json and makes an HTTP call. Your code never changes — the same client works in both environments.
