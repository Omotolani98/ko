Kọ́ is a framework for building type-safe distributed systems with declarative infrastructure annotations. Inspired by Encore.dev, built for the JVM. Annotations replace YAML — your code is the source of truth.

Why Kọ́?

Everything you need to go from code to production with zero infrastructure boilerplate.

◇

Declarative Infrastructure

Annotate what you need — databases, caches, topics — and the framework provisions it. No Terraform or Helm for local dev.

◍

Compile-Time Safety

The annotation processor validates cron expressions, API paths, naming conventions, and circular dependencies at build time.

◎

Zero-Config Local Dev

One command spins up Postgres, Kafka, Redis — whatever your services declare. Testcontainers under the hood.

◆

Cloud-Agnostic

Provider interfaces abstract AWS, GCP, and self-hosted. Same code runs locally, in Docker, or across clouds.

⇆

Service-to-Service Calls

Generated type-safe clients route calls in-process locally and over HTTP in production. No boilerplate.

▣

Dev Dashboard

Architecture graph, API explorer, distributed tracing, and database browser — all auto-generated from your code.

A taste of Kọ́.

One class. One database. One topic. One cron job. Zero boilerplate.

UserService.java Java 
@KoService("user-service")
public class UserService {

    @KoDatabase(name = "users")
    private KoSQLDatabase db;

    @KoPubSub(topic = "user-registered")
    private KoTopic<UserRegisteredEvent> registeredTopic;

    @KoAPI(method = "POST", path = "/users")
    public UserResponse createUser(CreateUserRequest req) {
        User user = db.exec(conn ->
            conn.insertReturning(
                "INSERT INTO users (name, email) VALUES (?, ?)",
                req.name(), req.email()
            )
        );
        registeredTopic.publish(new UserRegisteredEvent(user.id()));
        return UserResponse.from(user);
    }

    @KoAPI(method = "GET", path = "/users/:id")
    public UserResponse getUser(@PathParam("id") String id) {
        return db.queryOne("SELECT * FROM users WHERE id = ?", id)
            .map(UserResponse::from)
            .orElseThrow(() -> KoError.notFound("User not found"));
    }
}
